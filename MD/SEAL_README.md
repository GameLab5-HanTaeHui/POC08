## 카메라 시점 정의

본 게임의 카메라 시점은 완전 수직 정수리 탑뷰가 아니다.

플레이 공간과 전투 구조는 탑다운 방식을 기반으로 하지만, 캐릭터와 적의 실루엣은 2D 사이드뷰 또는 쿼터뷰처럼 상체, 팔, 무기, 부위 방향성이 보이도록 표현한다.

즉, 게임 플레이는 탑다운 방식으로 진행되지만, 비주얼 표현은 비스듬한 탑뷰 / 쿼터뷰에 가깝다.

이 시점을 사용하는 이유는 다음과 같다.

- 열쇠 무기의 실루엣을 명확하게 보여주기 위해
- 정면 / 측면 / 후방 공략을 구분하기 위해
- 보스의 팔, 다리, 머리, 방패 같은 부위를 식별하기 위해
- 봉인 집행 시 열쇠를 꽂는 연출을 보여주기 위해
- 순수 정수리 탑뷰에서 발생하는 시각 정보 부족을 피하기 위해

```text
→ 봉인 집행
→ 행동 제한
→ 그로기
→ 코어 해제
→ 코어 봉인도 누적
→ 최종 봉인
```

# 3. 컨셉

플레이어는 열쇠를 사용하는 봉인 집행자다.

플레이어의 공격은 단순한 물리적 피해가 아니다. 플레이어가 사용하는 모든 공격은 봉인 의식의 일부이며, 적을 공격하는 행위 자체가 봉인을 누적시키는 과정이다.

적들은 신체 곳곳에 봉인 가능한 부위를 가지고 있다. 플레이어는 공격을 통해 적의 봉인을 진행하며, 봉인이 완성된 부위는 해당 부위와 연결된 행동 패턴이 제한된다.

보스는 일정 수 이상의 봉인이 유지되면 그로기 상태에 진입한다. 그로기 상태에서는 숨겨져 있던 코어가 활성화된다.

플레이어는 코어에 접근하여 열쇠로 코어의 봉인을 해제한다. 코어 해제가 완료되면 딜 페이즈가 시작된다.

딜 페이즈는 보스의 체력을 깎는 시간이 아니라, 코어 봉인도를 누적시키는 시간이다. 코어 봉인도가 100%가 되면 최종 봉인을 통해 보스를 처치한다.

# 5. 핵심 컨셉

플레이어는 열쇠를 사용하는 봉인 집행자다.

적들은 신체 곳곳에 봉인 가능한 부위를 가지고 있다. 이 부위들은 단순한 약점이 아니라, 적의 공격 패턴과 직접 연결된 구조다.

플레이어는 비스듬한 탑뷰 / 쿼터뷰 전장 안에서 적의 공격 범위를 피하고, 정면, 측면, 후방, 사각지대를 공략하여 특정 부위에 봉인을 진행한다.

공격이 적중하면 해당 부위의 봉인도가 증가한다. 봉인도가 100%에 도달하면 해당 부위는 봉인 가능 상태가 된다. 이 상태에서 플레이어는 부위에 접근하여 열쇠를 직접 꽂는 봉인 집행을 수행한다.

부위가 봉인되면 적은 특정 행동을 사용하지 못하거나, 패턴이 약화되거나, 공격 방식이 변화한다.

엘리트 몬스터와 미니보스는 간소화된 봉인 구조를 가진다. 보스는 복수의 부위 봉인, 그로기, 코어, 딜 페이즈, 최종 봉인 구조를 가진다.

# 6. 게임의 핵심 문장

플레이어는 열쇠를 사용하여 적을 공격하고, 적을 쓰러뜨리거나 부위를 무력화시킨다.

공격은 전투 행동이고, 봉인은 마무리 행동이다.

공격만으로는 적을 완전히 처치할 수 없다.

일반 몬스터와 엘리트 몬스터는 체력이 0이 되거나 다운되면 봉인 가능 상태에 진입한다.

플레이어는 봉인 가능 상태의 적에게 접근하여 S 키로 봉인 집행을 수행해야 한다.

봉인에 성공한 적은 완전히 처치되며, 봉인하지 못한 적은 일정 시간이 지나 부활할 수 있다.

봉인 집행 성공 후 일정 범위 안에 다른 봉인 가능 대상이 존재할 경우, 플레이어는 일섬처럼 빠르게 추적 이동하며 연쇄 봉인을 수행할 수 있다.

보스전에서는 부위 공격으로 부위를 무력화하고, 부위 봉인으로 패턴을 제한하며, 최종적으로 코어를 봉인해 보스를 처치한다.

# 7. 핵심 전투 루프

## 핵심 변경점

공격과 봉인은 분리된다.

```text
공격 = 적을 쓰러뜨리거나 부위를 무력화시키는 전투 행동
봉인 = 쓰러진 적을 완전히 처치하거나 부활을 막는 마무리 행동
```

플레이어는 열쇠 무기로 적을 공격한다.

하지만 공격만으로는 적을 완전히 처치할 수 없다.

적은 체력이 0이 되거나 특정 조건으로 무력화되면 봉인 가능 상태가 된다.

플레이어는 봉인 가능 상태의 적에게 접근하여 S 키로 봉인 집행을 수행해야 한다.

봉인에 성공한 적은 완전히 처치된다.

봉인하지 못한 적은 일정 시간이 지나 부활할 수 있다.

## 일반 몬스터

```text
적 발견
→ 열쇠 공격
→ 체력 감소
→ 적 다운
→ 봉인 가능 상태
→ S 봉인 집행
→ 완전 처치
```

봉인하지 못했을 경우:

```text
적 다운
→ 봉인 제한 시간 종료
→ 적 부활
```

일반 몬스터는 빠른 봉인 집행 대상이다.

일반 몬스터 봉인 시 불릿 타임은 사용하지 않는다.

## 엘리트 몬스터

```text
적 발견
→ 열쇠 공격
→ 체력 감소
→ 약식 그로기 또는 다운
→ 봉인 가능 상태
→ S 봉인 집행
→ 완전 처치
```

엘리트 몬스터도 일반 몬스터처럼 빠른 봉인 집행 대상이다.

일반 몬스터보다 강한 피드백은 제공하지만, 불릿 타임은 사용하지 않는다.

## 다수 몬스터 / 일섬 연쇄 봉인

```text
여러 적 공격
→ 복수의 적 다운
→ 복수의 봉인 가능 대상 발생
→ S 입력
→ 첫 대상 봉인
→ 주변 봉인 가능 대상 자동 탐색
→ 자동 추적 이동
→ 연쇄 봉인
→ 다수 처치
```

일섬 연쇄 봉인은 일반 몬스터와 엘리트 몬스터를 빠르게 마무리하기 위한 핵심 액션이다.

봉인 집행 성공 후 일정 범위 안에 다른 봉인 가능 대상이 존재할 경우, 플레이어는 해당 대상에게 자동으로 추적 이동하여 연쇄 봉인을 수행할 수 있다.

이 연쇄 봉인은 일섬처럼 빠르게 이어지는 봉인 액션으로 표현된다.

## 미니보스

```text
공격
→ 부위 체력 / 내구도 감소
→ 부위 무력화
→ 부위 봉인 가능 상태
→ S 봉인 집행
→ 부위 봉인
→ 약식 그로기
→ 본체 봉인 가능
→ 봉인 집행
→ 처치
```

미니보스의 부위 봉인은 일반 / 엘리트 봉인과 같은 빠른 템포로 처리한다.

다만 일반 몬스터보다 강한 봉인 이펙트와 짧은 슬로우를 사용할 수 있다.

## 보스

```text
보스 패턴 관찰
→ 공격 범위 회피
→ 부위 공격
→ 부위 체력 / 내구도 감소
→ 부위 무력화
→ 부위 봉인 가능 상태
→ S 봉인 집행
→ 부위 봉인 완성
→ 해당 부위 패턴 제한
→ 추가 부위 봉인 진행
→ 그로기 조건 달성
→ 그로기 상태 진입
→ 코어 활성화
→ 코어 접근
→ 코어 해제
→ 딜 페이즈 진입
→ 코어 공격
→ 코어 봉인도 증가
→ 최종 봉인 가능
→ 코어 봉인 집행
→ 보스 처치
```

보스 부위 봉인은 전투 중 여러 번 발생하는 공략 행동이다.

따라서 보스 부위 봉인에는 긴 불릿 타임을 사용하지 않는다.

보스 코어 봉인 또는 최종 봉인에만 확실한 불릿 타임 연출을 사용한다.

# 8. 플레이 감각

플레이어가 느껴야 하는 핵심 감각은 아래와 같다.

- 위험한 패턴 사이를 파고드는 긴장감
- 공격을 통해 봉인도를 누적시키는 성취감
- 봉인도가 완성된 부위를 찾아내는 판단력
- 봉인 집행 타이밍을 노리는 집중감
- 직접 열쇠를 꽂아 봉인을 완성하는 손맛
- 봉인 성공으로 적의 행동이 제한되는 체감
- 제한 시간 안에 연속 봉인을 이어가는 압박감
- 강력한 적을 무력화시키는 성취감
- 그로기 상태를 만드는 전술적 만족감
- 코어가 노출되는 보상감
- 코어를 해제하는 전환점의 쾌감
- 딜 페이즈에서 코어 봉인도를 몰아 올리는 해방감
- 전투 사이클을 반복하며 적을 공략하는 레이드 감각

# 9. 플레이어

## 플레이어의 역할

플레이어는 열쇠 무기로 적을 공격하여 쓰러뜨리고, 봉인 가능 상태가 된 대상에게 직접 접근해 열쇠를 꽂아 봉인을 완성하는 봉인 집행자다.

플레이어의 공격은 적을 무력화시키기 위한 전투 행동이며, 봉인 집행은 적을 완전히 처치하거나 보스 부위를 봉인하는 마무리 행동이다.

## 플레이어 기본 행동

| 행동 | 설명 |
|---|---|
| 이동 | 방향키를 이용한 8방향 자유 이동 |
| 대시 | 8방향 이동 방향에 맞춰 발동되는 회피 / 접근 대시 |
| 기본 공격 | A 입력. 열쇠를 이용한 약공격. 적의 체력 또는 부위 내구도 감소 |
| 강공격 | A 홀드 후 릴리즈. 특정 타이밍에 한방을 노리는 공격. 높은 체력 / 부위 내구도 피해 |
| 봉인 집행 | S 입력. 봉인 가능 상태의 대상에게 접근해 열쇠를 꽂고 완전 처치 또는 부위 봉인 완성 |
| 코어 해제 | S 입력. 그로기 상태의 코어에 열쇠를 꽂아 딜 페이즈 진입 |
| 코어 공격 | 딜 페이즈 중 코어를 공격하여 코어 봉인도 누적 |
| 상호작용 | E |
| 회피 / 취소 | Shift |
| 메뉴 | Esc |

Unity New Input System을 사용한다.

키보드 조작을 기본으로 하되, Input Action 구조를 통해 게임패드 입력도 자동 대응 가능하도록 설계한다.

## 방향 결정 방식

플레이어의 공격 방향은 마우스 위치가 아닌 현재 입력 중인 방향키를 기준으로 결정된다.

방향 입력이 없는 경우 마지막 이동 방향을 유지한다.

모든 공격과 대시는 8방향을 기준으로 동작한다.

# 10. 탑뷰 이동 / 전장 구조

## 시점 적용 방식

본 파트에서 말하는 탑뷰는 완전 수직 정수리 탑뷰가 아니다.

실제 플레이 공간은 위에서 내려다보는 탑다운 구조를 사용하지만, 캐릭터와 보스는 상체, 팔, 무기, 부위 방향성이 보이는 비스듬한 탑뷰 / 쿼터뷰 표현을 사용한다.

따라서 전투 시스템은 8방향 이동과 탑다운 판정을 기반으로 하되, 시각 표현은 사이드뷰 캐릭터처럼 실루엣이 읽히는 방식을 목표로 한다.

## 이동 구조

플레이어는 8방향 이동을 사용한다.

이동은 방향키를 사용하며, 공격 방향 또한 현재 입력 방향 또는 마지막 이동 방향을 기준으로 결정된다.

## 공격 방향

마우스 조준을 사용하지 않는다.

모든 공격은 이동 방향을 기준으로 발동된다.

- 기본 공격
- 강공격
- 대시
- 봉인 집행 방향

모두 8방향 기준으로 동작한다.

## 이동 공간

전장은 하나의 아레나만 존재하지 않는다.

플레이어는 일반 구역을 진행하며 다음 적들을 상대한다.

- 일반 몬스터
- 엘리트 몬스터
- 미니보스
- 보스

## 전장 구조

전장에는 다음 요소가 존재할 수 있다.

- 벽
- 장애물
- 좁은 통로
- 개방 구역
- 봉인 장치
- 함정
- 위험 구역

## 보스 아레나

보스전은 별도의 아레나 공간에서 진행된다.

주요 요소:

- 안전 지대
- 위험 범위
- 봉인 집행 공간
- 코어 공략 공간
- 외곽 벽
- 패턴 표시 영역

# 11. 적 캐릭터

## 적 구성

플레이어가 상대하는 적은 다음 네 종류로 구분된다.

- 일반 몬스터
- 엘리트 몬스터
- 미니보스
- 보스

각 적은 서로 다른 봉인 구조와 전투 목적을 가진다.

## 일반 몬스터

일반 몬스터는 가장 기본적인 전투 대상이다.

```text
공격
→ 봉인도 누적
→ 봉인도 임계치 도달
→ 처치
```

봉인 집행 단계를 가지지 않는다.

## 엘리트 몬스터

엘리트 몬스터는 일반 몬스터보다 강력한 적이다.

```text
공격
→ 봉인도 누적
→ 그로기
→ 약점 노출
→ 처치
```

간소화된 그로기 구조를 가진다.

## 미니보스

미니보스는 보스 시스템의 입문 단계 역할을 담당한다.

```text
공격
→ 봉인도 누적
→ 봉인 가능 상태
→ 봉인 집행
→ 부위 봉인
→ 약식 그로기
→ 약점 노출
→ 처치
```

## 보스

보스는 게임의 최종 전투 구조를 담당한다.

```text
공격 적중
→ 봉인도 누적
→ 봉인 가능 상태
→ 봉인 집행
→ 부위 봉인
→ 행동 제한
→ 연속 봉인
→ 그로기
→ 코어 활성화
→ 코어 해제
→ 딜 페이즈
→ 코어 봉인도 증가
→ 페이즈 전환
→ 반복
→ 코어 봉인도 100%
→ 최종 봉인
→ 보스 처치
```

## 보스 페이즈 전환

보스는 단일 전투 사이클로 처치되지 않는다.

페이즈 전환 시 아래 변화가 발생할 수 있다.

- 새로운 패턴 추가
- 기존 패턴 강화
- 봉인 집행 난이도 증가
- 봉인 유지 난이도 증가
- 반격 패턴 증가

## 페이즈 전환 충격파

보스가 다음 페이즈로 진입할 경우 거대한 충격파를 발생시킨다.

효과:

- 플레이어 넉백
- 차징 공격 취소
- 봉인 집행 취소
- 위치 리셋

## 페이즈 전환 봉인 초기화

보스가 다음 페이즈로 진입할 경우 현재 봉인된 부위는 모두 해제된다.

초기화 대상:

- 봉인 상태
- 부위 봉인도
- 봉인 가능 상태

단, 코어 봉인도는 초기화되지 않는다.

## 봉인 저항

보스는 동일한 부위에 반복적으로 봉인당할수록 봉인에 대한 저항을 획득한다.

예시:

```text
1회 봉인: 100%
2회 봉인: 80%
3회 봉인: 60%
4회 봉인: 40%
```

## 봉인 파괴 패턴

일부 보스는 플레이어가 완성한 봉인을 직접 파괴할 수 있다.

봉인 파괴는 페이즈 전환과는 별개의 보스 전용 패턴이다.

# 12. 열쇠

## 열쇠의 정의

열쇠는 플레이어가 사용하는 유일한 무기이자 봉인 도구이다.

플레이어의 모든 공격은 열쇠를 통해 이루어지며, 모든 공격은 적의 봉인도를 누적시키는 행위이다.

```text
공격 = 무력화
봉인 = 완전 처치 / 부위 봉인 / 부활 방지
```

## 열쇠의 역할

- 적 공격
- 적 무력화
- 부위 내구도 감소
- 봉인 집행
- 일섬 연쇄 봉인
- 코어 해제
- 최종 봉인

## 열쇠의 전투 구조

```text
열쇠 공격 적중
→ 봉인도 누적
→ 봉인도 100%
→ 봉인 가능 상태
→ 봉인 집행
→ 부위 봉인
→ 행동 제한
→ 그로기
→ 코어 해제
→ 딜 페이즈
→ 코어 봉인도 누적
→ 최종 봉인
```

## 열쇠의 기본 분류

- 검형 열쇠
- 둔기형 열쇠
- 창형 열쇠
- 쌍열쇠
- 사슬 열쇠

모든 열쇠는 "공격을 통해 봉인을 누적시킨다"라는 공통 규칙을 가진다.

## 열쇠 설계 원칙

열쇠별 차이는 다음 요소에서 발생한다.

- 봉인도 누적 방식
- 공격 범위
- 공격 속도
- 봉인 집행 난이도
- 전투 스타일

## 비주얼 설계 방향

열쇠는 지팡이 또는 스태프처럼 보이지 않는다.

검, 둔기, 창, 사슬 무기 같은 실루엣을 가지되, 동시에 열쇠의 형태적 정체성을 가져야 한다.

# 13. 자물쇠

## 자물쇠의 정의

자물쇠는 적의 신체에 생성되는 봉인 상태를 의미한다.

자물쇠는 플레이어의 장비가 아니다.

## 자물쇠의 역할

- 부위 봉인 상태 표시
- 행동 제한
- 패턴 제한
- 그로기 조건 구성
- 코어 활성 조건 구성

## 자물쇠 생성 과정

```text
공격 적중
→ 봉인도 누적
→ 봉인도 100%
→ 봉인 가능 상태
→ 봉인 집행
→ 자물쇠 생성
→ 부위 봉인
```

## 자물쇠 상태

| 상태 | 설명 |
|---|---|
| 미생성 | 아직 봉인이 진행되지 않은 상태 |
| 봉인 가능 | 봉인도 100% 달성, 플레이어가 봉인 집행 가능 |
| 봉인 | 열쇠를 꽂아 자물쇠를 생성한 상태 |
| 해제 | 봉인 지속 시간 종료, 특정 패턴, 페이즈 전환, 봉인 파괴에 의해 제거된 상태 |

## 자물쇠와 행동 제한

예시:

```text
오른팔 자물쇠 → 검 공격 사용 불가
방패 자물쇠 → 방어 패턴 사용 불가
다리 자물쇠 → 돌진 사용 불가
머리 자물쇠 → 특수 패턴 사용 불가
```

## 자물쇠와 코어

```text
자물쇠
→ 그로기
→ 코어 활성화
```

# 14. 봉인

## 봉인의 정의

봉인은 플레이어가 적을 완전히 처치하거나, 보스의 특정 부위를 제한하기 위해 수행하는 핵심 마무리 행동이다.

봉인은 공격과 분리된다.

공격은 적을 쓰러뜨리거나 부위를 무력화시키는 행동이고, 봉인은 그 결과 발생한 봉인 가능 상태를 확정하는 행동이다.

## 봉인 구조

```text
열쇠 공격
→ 적 체력 / 부위 내구도 감소
→ 적 다운 / 부위 무력화
→ 봉인 가능 상태
→ S 봉인 집행
→ 자물쇠 생성
→ 완전 처치 / 부위 봉인
```

## 봉인 가능 상태

봉인 가능 상태는 플레이어가 봉인 집행을 수행할 수 있는 상태이다.

기존의 봉인도 100%만을 의미하지 않는다.

대상에 따라 봉인 가능 상태의 진입 조건이 달라진다.

```text
일반 몬스터 체력 0
→ 봉인 가능

엘리트 몬스터 체력 0 또는 약식 그로기
→ 봉인 가능

미니보스 부위 무력화
→ 부위 봉인 가능

보스 부위 무력화
→ 부위 봉인 가능

보스 코어 봉인도 100%
→ 최종 봉인 가능
```

## 봉인 집행

플레이어는 봉인 가능 상태가 된 대상에게 접근하여 S 키를 입력한다.

플레이어는 열쇠를 대상에게 꽂아 봉인을 완성한다.

## 일반 봉인

일반 몬스터와 엘리트 몬스터에게 사용하는 빠른 봉인이다.

일반 봉인은 전투 흐름을 끊지 않아야 한다.

불릿 타임은 사용하지 않는다.

## 부위 봉인

미니보스 또는 보스의 부위에 사용하는 봉인이다.

부위 봉인은 전투 중 여러 번 발생하는 공략 행동이다.

따라서 긴 불릿 타임을 사용하지 않는다.

짧은 슬로우, 강한 이펙트, 패턴 제한 피드백 정도를 사용한다.

## 코어 봉인

보스 코어 또는 최종 코어에 사용하는 봉인이다.

코어 봉인은 보스를 확실하게 봉인한다는 연출적 의미를 가진다.

이때는 불릿 타임, 카메라 줌, 강한 사운드, 대형 자물쇠 / 사슬 이펙트를 사용한다.

## 봉인하지 못한 적의 부활

일반 몬스터와 엘리트 몬스터는 체력이 0이 되어 다운되더라도 봉인되지 않으면 일정 시간이 지나 부활할 수 있다.

```text
적 다운
→ 봉인 가능 시간 발생
→ 봉인 실패
→ 적 부활
```

이 구조는 봉인을 선택이 아닌 필수 마무리 행동으로 만든다.

## 일섬 연쇄 봉인

봉인 집행 성공 후 일정 범위 안에 다른 봉인 가능 대상이 존재할 경우, 플레이어는 해당 대상에게 자동으로 추적 이동하여 연쇄 봉인을 수행할 수 있다.

이 연쇄 봉인은 일섬처럼 빠르게 이어지는 봉인 액션으로 표현된다.

```text
S 입력
→ 첫 대상 봉인
→ 주변 봉인 가능 대상 탐색
→ 자동 추적 이동
→ 다음 대상 봉인
→ 반복
```

## 일섬 연쇄 조건

- 첫 대상 봉인 성공
- 일정 범위 안에 다른 봉인 가능 대상 존재
- 대상이 시야 또는 이동 가능한 경로 안에 존재
- 연쇄 횟수 제한 미초과

## 일섬 종료 조건

- 범위 안에 봉인 가능 대상 없음
- 연쇄 횟수 초과
- 플레이어 피격
- 보스 패턴에 의한 방해
- 봉인 타이밍 실패

## 봉인 집행 등급

봉인 집행은 대상에 따라 연출 강도가 다르다.

모든 봉인 집행에 불릿 타임을 적용하지 않는다.

전투 템포를 유지하기 위해 일반 몬스터, 엘리트, 보스 부위 봉인은 빠르게 처리하고, 보스 코어 또는 최종 봉인에만 강한 불릿 타임 연출을 사용한다.

### 1단계. 일반 봉인

대상:

- 일반 몬스터
- 엘리트 몬스터

특징:

- 빠른 봉인
- 짧은 히트스톱
- 불릿 타임 없음
- 연쇄 봉인 가능

목적:

전투 흐름을 끊지 않고 적을 빠르게 마무리한다.

### 2단계. 부위 봉인

대상:

- 미니보스 부위
- 보스 부위

특징:

- 빠른 봉인
- 짧은 슬로우 가능
- 강한 봉인 이펙트
- 불릿 타임은 사용하지 않거나 매우 짧게 사용

목적:

전투 중 특정 패턴을 제한하고 공략 성공을 피드백한다.

### 3단계. 코어 봉인

대상:

- 보스 코어
- 최종 코어

특징:

- 불릿 타임 사용
- 카메라 줌
- 강한 사운드 연출
- 열쇠 삽입 강조
- 자물쇠 / 사슬 대형 이펙트

목적:

보스를 확실하게 봉인한다는 연출적 의미를 전달한다.

## 봉인의 목적

- 적의 부활 방지
- 일반 / 엘리트 몬스터 완전 처치
- 미니보스 / 보스 부위 제한
- 보스 패턴 차단
- 그로기 유도
- 코어 활성화
- 최종 봉인

---

# 15. 해제

## 해제의 정의

해제는 현재 코어에만 사용되는 특수 행동이다.

플레이어는 그로기 상태에서 활성화된 코어에 접근하여 열쇠를 사용해 코어를 해제할 수 있다.

코어 해제가 완료되면 딜 페이즈가 시작된다.

## 해제 구조

```text
그로기
→ 코어 활성화
→ 코어 접근
→ 해제
→ 딜 페이즈
```

## 추후 확장 가능성

향후 성장 시스템이 추가될 경우 해제는 플레이어 능력을 개방하는 기능으로 확장될 수 있다.

예시:

- 대쉬 봉인 해제
- 강공격 봉인 해제
- 무기군 봉인 해제
- 특수 능력 해제

---

# 16. 그로기

## 그로기의 정의

그로기는 적이 일정 조건 이상의 봉인을 당했을 때 발생하는 무력화 상태이다.

## 일반 몬스터

일반 몬스터는 그로기 상태를 가지지 않는다.

## 엘리트 몬스터

엘리트 몬스터는 약식 그로기 상태를 가진다.

## 미니보스

미니보스는 봉인 집행 이후 약식 그로기 상태에 진입할 수 있다.

## 보스

보스는 정식 그로기 시스템을 가진다.

```text
부위 봉인
→ 연속 봉인
→ 그로기 조건 달성
→ 정식 그로기
→ 코어 활성화
→ 코어 해제
→ 딜 페이즈
```

## 그로기 조건

예시:

- 왼팔 + 오른팔 봉인
- 방패 + 오른팔 봉인
- 3개 부위 연속 봉인
- 특정 패턴 중 봉인 성공

---

# 17. 코어

## 코어의 정의

코어는 강력한 적의 내부에 존재하는 핵심 구조물이다.

코어는 평상시 외부에 노출되지 않으며, 그로기 상태에 진입해야만 활성화된다.

## 코어의 목적

- 딜 페이즈 진입
- 코어 봉인도 누적
- 페이즈 진행
- 최종 봉인

## 코어 활성화

```text
부위 봉인
→ 그로기
→ 코어 활성화
```

## 코어 해제

```text
코어 활성화
→ 코어 접근
→ 해제
→ 딜 페이즈 진입
```

## 코어 봉인도

코어는 별도의 봉인도를 가진다.

딜 페이즈 동안 플레이어는 코어를 공격하여 코어 봉인도를 누적시킬 수 있다.

```text
코어 봉인도 0%
→ 25%
→ 50%
→ 75%
→ 100%
→ 최종 봉인 가능
```

---

# 18. 딜 페이즈

## 딜 페이즈의 정의

딜 페이즈는 플레이어가 코어를 공격하여 코어 봉인도를 누적시키는 단계이다.

보스의 체력을 감소시키는 구간이 아니다.

## 딜 페이즈 진입

```text
부위 봉인
→ 그로기
→ 코어 활성화
→ 코어 해제
→ 딜 페이즈
```

## 코어 봉인도

코어 봉인도는 각 딜 페이즈를 거치며 누적된다.

코어 봉인도는 초기화되지 않는다.

### 일반 보스 예시

```text
1페이즈: 0% → 50%
2페이즈: 50% → 100%
→ 최종 봉인 가능
```

### 최종 보스 예시

```text
1페이즈: 0% → 33%
2페이즈: 33% → 66%
3페이즈: 66% → 100%
→ 최종 봉인 가능
```

## 딜 페이즈 종료

종료 시 다음 요소가 발생한다.

- 코어 재봉인
- 그로기 종료
- 충격파 발생
- 플레이어 밀쳐냄
- 부위 봉인 해제
- 부위 봉인도 초기화

## 강제 페이즈 전환

딜 페이즈 중 코어 봉인도가 다음 페이즈 진입 조건에 도달한 경우 현재 딜 페이즈는 즉시 종료된다.

## 페이즈 수 제한

일반 보스는 최대 2개의 페이즈를 가진다.

최종 보스는 예외적으로 3개의 페이즈를 가진다.

---

# 19. 페이즈 시스템

## 페이즈의 정의

페이즈는 보스 전투의 진행 단계를 의미한다.

## 페이즈 진행 구조

```text
부위 봉인
→ 그로기
→ 코어 활성화
→ 코어 해제
→ 딜 페이즈
→ 코어 봉인도 누적
→ 페이즈 전환
→ 반복
→ 최종 봉인
```

## 일반 보스

```text
1페이즈
→ 2페이즈
→ 최종 봉인
→ 처치
```

## 최종 보스

```text
1페이즈
→ 2페이즈
→ 3페이즈
→ 최종 봉인
→ 처치
```

## 페이즈 전환 충격파

페이즈 전환 시 보스는 거대한 충격파를 발생시킨다.

## 봉인 초기화

페이즈 전환 시 현재 봉인된 모든 부위는 해제된다.

코어 봉인도는 초기화되지 않는다.

## 봉인 저항

보스는 반복적으로 같은 부위를 봉인당할 경우 해당 부위에 대한 저항을 획득한다.

## 봉인 파괴 패턴

일부 보스는 플레이어가 완성한 봉인을 직접 파괴할 수 있다.

---

# 20. 패턴 설계 원칙

## 패턴의 정의

패턴은 보스가 플레이어를 공격하거나 플레이어의 봉인 진행을 방해하기 위해 사용하는 행동이다.

## 패턴 설계 목표

패턴의 목적은 플레이어를 죽이는 것이 아니다.

패턴의 목적은 플레이어가 위험을 피하며 봉인을 수행하도록 만드는 것이다.

좋은 패턴:

```text
공략 가능
→ 부위 접근 가능
→ 봉인 가능
```

## 패턴 분류

- 일반 공격 패턴
- 제어 패턴
- 방어 패턴
- 봉인 대응 패턴

## 패턴과 부위의 연결

모든 주요 패턴은 특정 부위와 연결될 수 있다.

예시:

```text
오른팔 → 검 공격
다리 → 돌진
머리 → 광역 패턴
```

---

# 21. 부위 설계

## 부위의 정의

부위는 보스의 행동을 구성하는 핵심 구조물이다.

각 부위는 하나 이상의 패턴과 연결된다.

## 부위 유형

- 공격 부위
- 이동 부위
- 방어 부위
- 특수 부위

## 부위 수 설계

| 적 유형 | 권장 부위 수 |
|---|---|
| 엘리트 | 1~2개 |
| 미니보스 | 2~4개 |
| 일반 보스 | 3~6개 |
| 최종 보스 | 6개 이상 |

---

# 22. 봉인 가능 상태 / 내구도 설계

## 기존 봉인도 구조 변경

기존의 `공격 = 봉인도 누적` 구조는 사용하지 않는다.

공격과 봉인을 분리한다.

```text
공격 = 체력 / 부위 내구도 감소
봉인 = 봉인 가능 상태에서 수행하는 마무리 행동
```

## 일반 몬스터

일반 몬스터는 체력이 0이 되면 다운 상태가 된다.

다운 상태는 봉인 가능 상태이다.

```text
체력 0
→ 다운
→ 봉인 가능
```

봉인하지 않으면 일정 시간 후 부활할 수 있다.

## 엘리트 몬스터

엘리트 몬스터는 체력이 0이 되거나 약식 그로기에 진입하면 봉인 가능 상태가 된다.

일반 몬스터보다 봉인 가능 시간이 짧거나, 부활 속도가 빠를 수 있다.

## 미니보스 부위

미니보스는 특정 부위 내구도가 0이 되면 해당 부위가 봉인 가능 상태가 된다.

부위 봉인 성공 시 특정 패턴이 제한된다.

## 보스 부위

보스는 부위별 내구도를 가진다.

부위 내구도가 0이 되면 해당 부위는 봉인 가능 상태가 된다.

```text
부위 공격
→ 부위 내구도 감소
→ 부위 무력화
→ 부위 봉인 가능
→ S 봉인 집행
→ 부위 봉인
→ 연결 패턴 제한
```

## 코어 봉인도

코어는 기존처럼 별도의 봉인도를 가진다.

코어 봉인도는 딜 페이즈에서만 누적된다.

코어 봉인도가 100%가 되면 최종 봉인 가능 상태가 된다.

```text
딜 페이즈
→ 코어 공격
→ 코어 봉인도 증가
→ 코어 봉인도 100%
→ 최종 봉인 가능
```

## UI 표시 원칙

일반 / 엘리트 몬스터는 체력 또는 다운 상태를 중심으로 표시한다.

미니보스 / 보스 부위는 부위 내구도 또는 부위 무력화 상태를 표시한다.

코어는 코어 봉인도를 표시한다.

## 설계 의도

공격은 액션 전투의 기본 흐름을 담당한다.

봉인은 적을 완전히 끝내는 마무리 행동을 담당한다.

이 구조를 통해 봉인은 선택적 추가 행동이 아니라, 적의 부활을 막고 전투를 종결시키는 필수 행동이 된다.

---

# 23. 보스 설계 원칙

## 보스의 정의

보스는 플레이어가 봉인 시스템을 가장 깊게 경험하는 대상이다.

## 보스 전투 구조

```text
패턴 분석
→ 부위 공략
→ 봉인도 누적
→ 봉인 집행
→ 부위 봉인
→ 그로기
→ 코어 활성화
→ 코어 해제
→ 딜 페이즈
→ 코어 봉인도 증가
→ 페이즈 전환
→ 반복
→ 최종 봉인
→ 처치
```

## 보스 설계 목표

좋은 보스는 플레이어가 봉인을 수행하도록 유도해야 한다.

## 페이즈 설계

일반 보스는 최대 2페이즈를 가진다.

최종 보스만 3페이즈를 가진다.

---

# 24. 플레이어 성장 시스템

## 성장의 정의

플레이어는 전투를 통해 새로운 열쇠와 능력을 해제할 수 있다.

성장 방식은 수치 증가보다 능력 개방에 집중한다.

## 어빌리티 열쇠

어빌리티 열쇠는 플레이어의 봉인된 능력을 해제하는 특수 열쇠이다.

```text
어빌리티 열쇠
→ 능력 해제
→ 플레이 스타일 변화
```

## 영구 성장

일부 어빌리티 열쇠는 영구 해금 형태로 설계할 수 있다.

예시:

- 신규 열쇠 해금
- 신규 시작 옵션
- 신규 무기 계보 해금
- 신규 어빌리티 해금

보스 해금과 층 개방은 현재 제외한다.

---

# 25. 일반 몬스터 설계

일반 몬스터는 플레이어가 가장 자주 상대하는 적이다.

전투 구조:

```text
공격
→ 봉인도 누적
→ 봉인도 임계치 도달
→ 처치
```

봉인 집행은 존재하지 않는다.

일반 몬스터는 1~3개의 단순 패턴을 가진다.

유형:

- 근접형
- 원거리형
- 돌격형
- 지원형

---

# 26. 엘리트 몬스터 설계

엘리트 몬스터는 일반 몬스터와 보스 사이를 연결하는 적이다.

전투 구조:

```text
공격
→ 봉인도 누적
→ 그로기 조건 달성
→ 약식 그로기
→ 약점 노출
→ 처치
```

엘리트는 봉인 집행을 요구하지 않는다.

권장 패턴 수: 3~5개

---

# 27. 미니보스 설계

미니보스는 보스 시스템의 입문 단계 역할을 수행하는 적이다.

플레이어는 미니보스를 통해 처음으로 봉인 집행을 경험한다.

전투 구조:

```text
공격
→ 봉인도 누적
→ 봉인도 100%
→ 봉인 집행
→ 부위 봉인
→ 약식 그로기
→ 처치
```

권장 부위 수: 2~4개

권장 패턴 수: 4~6개

---

# 28. 보스 설계

보스는 게임의 핵심 콘텐츠이다.

## 일반 보스 구조

일반 보스는 최대 2페이즈를 가진다.

## 최종 보스 구조

최종 보스만 3페이즈를 가진다.

## 보스 필수 구성 요소

- 봉인 가능한 부위
- 그로기 조건
- 코어
- 코어 해제
- 딜 페이즈
- 최종 봉인

## 보스 아키타입

### 기사형 보스

- 근접 중심
- 방향 공략 강조
- 방패 활용

### 거수형 보스

- 넓은 공격 범위
- 공간 압박
- 거대한 신체

### 비행형 보스

- 높은 기동성
- 위치 변화
- 탄막 패턴

비행형 보스는 탄막을 이용하여 플레이어의 이동 경로를 제한한다.

### 기계형 보스

- 정밀한 공격
- 다수의 모듈
- 높은 봉인 저항
- 탄막 패턴

## 탄막 설계 원칙

탄막은 플레이어를 즉시 처치하기 위한 시스템이 아니다.

탄막은 플레이어의 이동 경로를 제한하고 특정 부위를 공략하기 어렵게 만드는 장치이다.

---

# 29. 보상 시스템

보상은 플레이어가 전투를 통해 획득하는 성장 자원이다.

보상은 단순한 수치 증가보다 플레이 스타일 변화를 우선한다.

## 보상 획득 구조

```text
전투 승리
→ 보상 획득
→ 빌드 구성
→ 다음 전투
→ 반복
```

## 보상 종류

- 일반 몬스터: 열쇠 조각, 소량의 자원
- 엘리트: 어빌리티 열쇠 선택권, 희귀 재화
- 미니보스: 희귀 열쇠, 특수 능력 해금, 열쇠 개조 부품
- 보스: 어빌리티 열쇠, 신규 열쇠 계보, 특수 패시브

---

# 30. 던전 구조

던전은 플레이어가 보스를 봉인하기 위해 진입하는 전투 구역이다.

던전은 고정 직선 진행보다 노드 기반 선택 구조를 가진다.

플레이어는 미니맵을 펼쳐 다음 갈 지역을 선택한다.

```text
층 시작
→ 다음 경로 선택
→ 전투
→ 다음 경로 선택
→ 전투
→ 다음 경로 선택
→ 보스
```

노드 종류:

- 일반 전투
- 엘리트
- 미니보스
- 쉼터
- 상점
- 이벤트
- 보스

---

# 31. 층 구조

층은 하나의 보스를 봉인하기 위한 전투 구간이다.

## 기본 진행 구조

```text
시작 구역
→ 1. 일반 전투
→ 2. 일반 전투
→ 엘리트 또는 미니보스
→ 쉼터
→ 3. 일반 전투
→ 보스
→ 다음 층
```

## 노드 선택 구조

플레이어는 미니맵을 펼쳐 몇 갈래의 루트 중 하나를 선택하고 앞으로 나아간다.

## 엘리트 / 미니보스 구역

엘리트와 미니보스는 동일 선상에 둔다.

| 구분 | 확률 |
|---|---:|
| 엘리트 | 90% |
| 미니보스 | 10% |

엘리트는 높은 보상을 주고, 미니보스는 더 좋은 보상을 준다.

## 쉼터

쉼터에서는 체력 회복, 보상 선택, 어빌리티 선택, 열쇠 정비 등을 수행할 수 있다.

# 32. 게임 루프

## 전체 게임 루프

```text
층 진입
→ 노드 선택
→ 전투
→ 보상 획득
→ 어빌리티 획득
→ 다음 노드 선택
→ 보스 도달
→ 보스 봉인
→ 다음 층 진입
→ 반복
```

## 보스 전투 루프

```text
패턴 분석
→ 부위 공략
→ 봉인도 누적
→ 봉인 집행
→ 부위 봉인
→ 그로기
→ 코어 활성화
→ 코어 해제
→ 딜 페이즈
→ 코어 봉인도 증가
→ 페이즈 전환
→ 반복
→ 최종 봉인
→ 처치
```

# 33. 승리 조건 및 실패 조건

## 목표

플레이어의 최종 목표는 최종 보스를 봉인하는 것이다.

## 층 클리어 조건

해당 층의 보스를 봉인하면 층이 클리어된다.

## 런 클리어 조건

최종 층의 보스를 봉인하면 현재 런이 종료된다.

## 패배 조건

플레이어의 체력이 0이 되면 현재 런은 실패한다.

## 영구 진행 요소

유지 가능 요소:

- 신규 열쇠 해금
- 신규 시작 옵션
- 신규 무기 계보 해금
- 신규 어빌리티 해금

보스 해금과 층 개방은 제외한다.

# 34. 핵심 설계 원칙

## 원칙 1. 봉인이 전투의 중심이다

플레이어는 적을 공격하기 위해 존재하지 않는다.

플레이어는 적을 봉인하기 위해 존재한다.

## 원칙 2. 패턴 회피는 수단이다

회피는 봉인을 수행하기 위한 위치 확보 수단이다.

## 원칙 3. 부위 봉인은 전투를 변화시켜야 한다

봉인 전과 봉인 후의 전투는 명확히 달라야 한다.

## 원칙 4. 그로기는 보상 단계이다

그로기는 전투 종료 상태가 아니다.

## 원칙 5. 딜 페이즈는 체력 감소 구간이 아니다

딜 페이즈의 목적은 코어 봉인도 누적이다.

## 원칙 6. 탄막은 공략을 방해하는 장치이다

탄막은 위치 제한, 부위 접근 방해, 봉인 방해 역할을 수행한다.

## 원칙 7. 성장은 선택지를 늘려야 한다

성장은 공격력 증가보다 플레이 스타일 변화에 집중한다.

## 원칙 8. 모든 보스는 공략 가능해야 한다

좋은 보스는 패턴을 이해할 수 있고, 부위를 발견할 수 있으며, 봉인할 수 있어야 한다.

## 원칙 9. 반복 플레이는 빌드 변화에서 나온다

반복 플레이의 목적은 새로운 열쇠, 어빌리티, 플레이 스타일을 경험하는 것이다.

# 35. 열쇠 무기 시스템

열쇠는 플레이어의 주 무기이며 봉인을 수행하기 위한 도구이다.

## 기본 구조

- 앞부분 / 비트: 공격 방식 결정
- 몸체 / 샤프트: 사거리 결정
- 손잡이 / 헤드: 특수 효과 결정

## 근접 열쇠

### 검형 열쇠

- 균형형
- 빠른 공격
- 안정적인 봉인

### 둔기형 열쇠

- 느림
- 높은 봉인도 획득량
- 강력한 강공격

### 창형 열쇠

- 긴 사거리
- 정면 공략 특화

### 쌍열쇠

- 높은 공격속도
- 낮은 개별 봉인 효율
- 연타 중심

## 기본 공격

입력: A

## 강공격

입력:

```text
A 홀드
→ 릴리즈
```

## 봉인 집행

입력: S

조건:

- 봉인도 100%

## 코어 해제

입력: S

조건:

- 코어 활성화

# 36. 열쇠 계보 시스템

열쇠 계보는 플레이어가 사용하는 열쇠 무기의 계열을 의미한다.

각 계보는 고유한 전투 철학을 가진다.

## 기사 계보

전투 철학: 균형

```text
녹슨 열쇠
→ 기사 열쇠
→ 수호 기사 열쇠
→ 왕실 기사 열쇠
```

## 감옥 계보

전투 철학: 강제 봉인

```text
녹슨 열쇠
→ 감옥 열쇠
→ 처형 열쇠
→ 봉인 집행자 열쇠
```

## 심판 계보

전투 철학: 코어 공략

```text
녹슨 열쇠
→ 심판 열쇠
→ 성역 열쇠
→ 최종 심판 열쇠
```

## 추적자 계보

전투 철학: 기동 공략

```text
녹슨 열쇠
→ 추적자 열쇠
→ 사냥꾼 열쇠
→ 그림자 열쇠
```

## 감시자 계보

전투 철학: 거리 유지

```text
녹슨 열쇠
→ 감시자 열쇠
→ 탑지기 열쇠
→ 영원의 감시자 열쇠
```

# 37. 어빌리티 열쇠 시스템

어빌리티 열쇠는 플레이어 자신의 봉인된 능력을 해제하는 특수 열쇠이다.

## 이동 계열

- 대쉬 열쇠: 대쉬 2회 충전
- 추진 열쇠: 대쉬 거리 증가
- 그림자 열쇠: 대쉬 무적 시간 증가
- 질주 열쇠: 대쉬 후 이동속도 증가

## 봉인 계열

- 공명 열쇠: 봉인도 획득량 증가
- 집중 열쇠: 강공격 봉인도 증가
- 집행 열쇠: 봉인 집행 속도 증가
- 유지 열쇠: 부위 봉인 지속시간 증가

## 코어 계열

- 해제 열쇠: 코어 해제 속도 증가
- 심판 열쇠: 코어 봉인도 획득량 증가
- 공허 열쇠: 딜 페이즈 지속시간 증가
- 최종 열쇠: 최종 봉인 속도 증가

## 전투 계열

- 연속 열쇠: 기본 공격 추가타 발생
- 관통 열쇠: 공격 범위 증가
- 균열 열쇠: 강공격 충격파 발생
- 폭쇄 열쇠: 강공격 적중 시 범위 봉인도 획득

## 특수 계열

- 복제 열쇠: 다음 어빌리티 선택 시 추가 선택지 생성
- 운명 열쇠: 희귀 어빌리티 등장 확률 증가
- 반전 열쇠: 봉인 집행 성공 시 대쉬 충전 회복
- 연결 열쇠: 하나의 부위를 봉인하면 다른 부위에도 일부 봉인도 전이

# 38. UI / UX 설계

## 목적

UI는 플레이어에게 현재 전투 상태를 명확하게 전달해야 한다.

## 플레이어 UI

- 체력
- 대쉬 충전 수
- 현재 열쇠
- 현재 어빌리티

## 보스 UI

- 현재 페이즈
- 부위 상태
- 코어 상태
- 그로기 상태

## 부위 봉인도 UI

플레이어가 공격 중인 부위만 표시한다.

예시:

```text
오른팔 [██████░░░░] 60%
머리   [██░░░░░░░░] 20%
```

## 봉인 가능 상태 UI

예시:

- 자물쇠 아이콘 생성
- S 아이콘 표시
- 부위 외곽선 강조

## 공격 성공 피드백

플레이어와 적 모두 공격 성공 시 즉시 피드백을 제공해야 한다.

## 플레이어 공격 피드백

시각:

- 히트 이펙트
- 무기 궤적 강조
- 적 피격 플래시
- 봉인 파편 생성

연출:

- 히트 스톱
- 카메라 흔들림
- 적 경직

사운드:

- 타격음
- 금속 충돌음
- 봉인음

## 적 공격 피드백

시각:

- 화면 가장자리 플래시
- 피격 방향 표시

연출:

- 플레이어 경직
- 카메라 흔들림
- 짧은 슬로우

## 봉인 가능 상태 피드백

게이지 + 부위 시각 변화를 함께 사용한다.

일반 / 엘리트는 체력이 0이 되거나 다운되면 봉인 가능 표시를 띄운다.

미니보스 / 보스 부위는 부위 내구도가 0이 되면 봉인 가능 표시를 띄운다.

코어는 코어 봉인도 100%가 되면 최종 봉인 가능 표시를 띄운다.

```text
일반 / 엘리트 다운
→ 봉인 가능 아이콘

부위 무력화
→ 부위 외곽선 강조
→ S 아이콘

코어 봉인도 100%
→ 최종 봉인 연출 준비
```

## 봉인 집행 피드백

```text
집행 시작
→ 슬로우 모션
→ 열쇠 삽입
→ 자물쇠 생성
→ 사슬 전개
→ 부위 봉인 완료
```

## 공격 성공 / 실패 구분

공격 성공:

- 히트 스톱 발생
- 봉인 파편 발생
- 타격음 발생

공격 실패:

- 히트 스톱 없음
- 봉인 파편 없음
- 허공 베기 사운드

필수:

- 공략 가능

필수:

필수:

# 40. 프로젝트 아키텍처

## 목적

실제 구현 단계에서 필요한 관리자 구조와 시스템 책임 분리를 정의한다.

기획 시스템이 변경되어도 아키텍처는 최대한 유지 가능해야 한다.

## 핵심 관리자 구조

| 관리자 | 역할 |
|---|---|
| GameManager | 게임 전체 상태 관리 |
| DungeonManager | 런, 층, 노드 진행 관리 |
| StageManager | 현재 스테이지 / 전투 구역 관리 |
| BattleManager | 전투 시작, 종료, 보상 진입 관리 |
| PlayerManager | 플레이어 생성, 상태, 참조 관리 |
| EnemyManager | 일반 몬스터, 엘리트, 미니보스 생성 관리 |
| BossManager | 보스 전투 전용 흐름 관리 |
| UIManager | UI 화면 전환 및 주요 UI 참조 관리 |
| AudioManager | BGM, SFX 관리 |
| SaveManager | 영구 해금 및 설정 저장 관리 |
| PoolManager | 투사체, 이펙트, 몬스터 풀링 관리 |
| EventManager / EventBus | 시스템 간 이벤트 전달 관리 |

## GameManager

게임 전체 상태를 관리한다.

주요 상태:

```text
Title
MainMenu
Hub
RunStart
Dungeon
Battle
Reward
Pause
GameOver
RunClear
```

## DungeonManager

- 현재 층 정보 관리
- 노드 맵 생성
- 다음 노드 선택 처리
- 보스 노드 도달 여부 판단
- 런 진행도 관리

## StageManager

- 전투 공간 생성
- 적 배치
- 플레이어 위치 세팅
- 스테이지 클리어 판정
- 다음 구역 이동 준비

## BattleManager

- 전투 시작
- 적 웨이브 관리
- 전투 종료 판정
- 보상 화면 호출
- 플레이어 사망 처리

## BossManager

# 41. StateMachine 구조

## 목적

플레이어, 적, 보스는 상태가 많고 전환 조건이 명확하다.

따라서 StateMachine 구조를 사용하여 행동을 분리한다.

## StateMachine 기본 구조

```csharp
public interface IState
{
    void Enter();
    void Update();
    void FixedUpdate();
    void Exit();
}
```

```csharp
public class StateMachine
{
    private IState _currentState;

    public void ChangeState(IState newState)
    {
        _currentState?.Exit();
        _currentState = newState;
        _currentState?.Enter();
    }

    public void Update()
    {
        _currentState?.Update();
    }

    public void FixedUpdate()
    {
        _currentState?.FixedUpdate();
    }
}
```

## Player StateMachine

권장 상태:

```text
PlayerIdleState
PlayerMoveState
PlayerDashState
PlayerAttackState
PlayerChargeAttackState
PlayerSealExecutionState
PlayerCoreUnlockState
PlayerChainSealState
PlayerHitState
PlayerDeadState
```

상태 전환 예시:

```text
Idle
→ Move
→ Attack
→ Dash
→ SealExecution
→ CoreUnlock
→ Hit
→ Dead
```

## 상태 분리 원칙

상태 클래스는 하나의 행동 책임만 가진다.

나쁜 구조:

```csharp
void Update()
{
    if (isDead) { ... }
    else if (isDash) { ... }
    else if (isAttack) { ... }
    else if (isSeal) { ... }
}
```

StateMachine을 사용하여 상태 전환을 명확하게 분리한다.

- 씬에서 오브젝트를 찾기 쉽게 이름을 명확하게 작성한다.
